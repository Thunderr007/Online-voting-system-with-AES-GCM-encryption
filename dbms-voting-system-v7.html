How persistence works

  1. Every important change updates a JavaScript state object.
  2. That state is saved in localStorage under one application key.
  3. When the HTML opens again, the app reads the stored JSON and restores users, votes, candidates, admin info, and the last logged-in user.
  4. This works only in the same browser on the same device because local storage is browser-based, not database-based.
  5. For a real DBMS project, move this state to MySQL or PostgreSQL through backend APIs.

WebOTP API integration

  1. This app uses the WebOTP API (navigator.credentials.get({ otp: { transport: ['sms'] } })) to auto-fill OTP codes from incoming SMS messages.
  2. When registration triggers OTP generation, a navigator.credentials.get() call is started in the background with an AbortController.
  3. On supported browsers (Chrome on Android), when the SMS arrives containing the OTP and the page's domain on the last line (@yourdomain.com #123456), the browser prompts the user to allow auto-fill.
  4. If the user consents, the OTP is automatically filled into the input field.
  5. The listener is automatically cancelled if the user submits manually, or after 5 minutes (matching OTP expiry).
  6. The OTP input uses autocomplete="one-time-code" as a fallback — Safari will suggest the OTP from SMS keyboard suggestions even without full WebOTP support.
  7. In production, your SMS message must end with: @yourdomain.com #123456 for WebOTP to trigger.

Production OTP steps

  1. Create backend routes for registration, OTP send, OTP verify, login, vote, export, and verify.
  2. Store hashed OTP values and expiry times in the users or otp_logs table.
  3. Send SMS OTP using Twilio, MSG91, Fast2SMS, or Firebase Auth — format the SMS with @yourdomain.com #OTP on the last line for WebOTP.
  4. Store votes and audit data in SQL, not in local storage.
  5. Use server sessions or JWT after login and role checks for admin-only pages.

SQL schema and DBMS logic

-- ============================================================
-- VoteVault DBMS Schema  (reflects full application state)
-- ============================================================

-- Stores all registered voters and admins.
-- UUIDs are used as primary keys to match the JS crypto.randomUUID() used in the app.
CREATE TABLE users (
  user_id       CHAR(36)     PRIMARY KEY,          -- crypto.randomUUID()
  full_name     VARCHAR(120) NOT NULL,
  phone_number  VARCHAR(15)  NOT NULL UNIQUE,
  voter_id      CHAR(8)      NOT NULL UNIQUE,
  role          ENUM('voter','admin') DEFAULT 'voter',
  is_verified   BOOLEAN      DEFAULT FALSE,         -- set TRUE after OTP verification
  has_voted     BOOLEAN      DEFAULT FALSE,         -- set TRUE after castVote()
  created_at    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);

-- Temporary OTP requests created during registration.
-- One pending OTP per session; row is deleted once verified or expired.
CREATE TABLE otp_pending (
  otp_id        INT          PRIMARY KEY AUTO_INCREMENT,
  full_name     VARCHAR(120) NOT NULL,
  phone_number  VARCHAR(15)  NOT NULL,
  voter_id      CHAR(8)      NOT NULL,
  role          ENUM('voter','admin') DEFAULT 'voter',
  otp_hash      VARCHAR(255) NOT NULL,              -- store hashed OTP, not plaintext
  expires_at    DATETIME     NOT NULL,              -- OTP valid for 5 minutes
  created_at    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);

-- Stores election candidates.
CREATE TABLE candidates (
  candidate_id    CHAR(36)     PRIMARY KEY,         -- crypto.randomUUID()
  candidate_name  VARCHAR(120) NOT NULL,
  created_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);

-- Each voter may cast exactly one vote (enforced by UNIQUE on user_id).
-- Vote payload is AES-GCM encrypted; integrity verified by SHA-256 hash.
CREATE TABLE votes (
  vote_id         CHAR(36)     PRIMARY KEY,         -- crypto.randomUUID()
  user_id         CHAR(36)     NOT NULL UNIQUE,     -- one vote per user
  candidate_id    CHAR(36)     NOT NULL,
  encrypted_vote  LONGTEXT     NOT NULL,            -- JSON: { iv, salt, data } all base64
  vote_hash       CHAR(64)     NOT NULL,            -- SHA-256 of plaintext vote payload
  created_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id)      REFERENCES users(user_id),
  FOREIGN KEY (candidate_id) REFERENCES candidates(candidate_id)
);

-- Tracks every encrypted export file created by an admin.
-- Only the first-registered admin can decrypt exports (first_admin_id enforced in app logic).
CREATE TABLE export_audit (
  export_id       INT          PRIMARY KEY AUTO_INCREMENT,
  exported_by     CHAR(36)     NOT NULL,            -- user_id of exporting admin
  export_hash     CHAR(64)     NOT NULL,            -- SHA-256 of the plaintext export JSON
  encrypted_blob  LONGTEXT     NOT NULL,            -- AES-GCM encrypted export payload
  created_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (exported_by) REFERENCES users(user_id)
);

-- Stores app-level configuration (passphrase ref, first admin pointer, etc.)
-- Single-row table; insert once on first admin registration.
CREATE TABLE app_config (
  config_id         INT       PRIMARY KEY AUTO_INCREMENT,
  first_admin_id    CHAR(36)  DEFAULT NULL,         -- user_id of first registered admin
  export_passphrase VARCHAR(255) NOT NULL,          -- hashed passphrase for export encryption
  created_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (first_admin_id) REFERENCES users(user_id)
);

-- ============================================================
-- Useful views
-- ============================================================

-- Live vote tally per candidate
CREATE VIEW vote_results AS
  SELECT c.candidate_id, c.candidate_name,
         COUNT(v.vote_id) AS total_votes
  FROM candidates c
  LEFT JOIN votes v ON v.candidate_id = c.candidate_id
  GROUP BY c.candidate_id, c.candidate_name
  ORDER BY total_votes DESC;

-- Full user status overview (mirrors the DB View panel)
CREATE VIEW user_status AS
  SELECT user_id, full_name, phone_number, voter_id,
         role, is_verified, has_voted, created_at
  FROM users
  ORDER BY created_at ASC;
